Web scraping con Playwright

El scraping consiste en extraer datos de sitios web de forma automatica. El desafio moderno es que la mayoria de sitios usan JavaScript para renderizar su contenido, por lo que un simple requests.get() devuelve una pagina vacia.

Por que Playwright y no requests?

MercadoLibre es una Single Page Application (SPA): la lista de propiedades se carga via JavaScript despues de que la pagina inicial llega al navegador. Playwright lanza un navegador Chromium real y espera a que el contenido aparezca.

Playwright: conceptos clave

ConceptoQue es
async_playwright()Context manager que gestiona el ciclo de vida de Playwright
browserInstancia del proceso Chromium (headless por defecto)
contextPerfil de navegacion aislado (cookies, localStorage propios)
pageUna pestaña del navegador. Sobre esta se llama goto(), content(), etc.
🧐
Por que crear un context por cada detalle? Los contexts son completamente aislados: cookies, sesiones, headers. Scrapeando detalles en paralelo, queremos que cada pestaña sea independiente para evitar que el sitio detecte el patron de un solo "usuario" abriendo 6 tabs simultaneas con las mismas cookies.

Estructura async del scraper

scraper/scraper.py — estructura principal 
import asyncio
from playwright.async_api import async_playwright

async def run():
    async with async_playwright() as p:
        # Lanzamos Chromium en modo headless (sin ventana visible)
        browser = await p.chromium.launch(headless=True)

        # Fase 1: scrapeamos listados de busqueda
        listings = await scrape_all_listings(browser)

        # Fase 2: scrapeamos detalles en paralelo

        await browser.close()

    # Procesamos y guardamos
    apartments = build_all_apartments(listings, details)
    save_results(apartments)

# Punto de entrada
if __name__ == "__main__":
    asyncio.run(run())

Navegar con resiliencia: safe_goto

Los sitios de scraping a veces dan timeout o errores de red. En vez de dejar que el scraper falle completamente, implementamos reintentos:

scraper/scraper.py 
async def safe_goto(page: Page, url: str, retries: int = 2) -> bool:
    """Navega a una URL con reintentos. Devuelve True si tuvo exito."""
    for attempt in range(retries):
        try:
            # wait_until="networkidle": espera hasta que no haya requests de red
            # Esto asegura que el JS ya cargo todos los datos
            await page.goto(url, wait_until="networkidle", timeout=60000)
            return True
        except Exception:
            log.warning(f"Intento {attempt+1} fallido. Reintentando...")
            # Pausa aleatoria para no sobrecargar el servidor
            await asyncio.sleep(random.uniform(4, 8))
    return False  # Todos los reintentos fallaron
⚠
Sobre el scraping etico Agrega pausas entre requests (asyncio.sleep) para no sobrecargar los servidores. Limita la concurrencia. Revisa el robots.txt del sitio. El scraping excesivo puede resultar en bloqueo de IP.

Estrategia de paginacion

MercadoLibre usa offset para paginar. La pagina 1 empieza en 0, la pagina 2 en 49, etc:

scraper/scraper.py 
async def scrape_zone(page, zone_name, zone_slug, seen_urls, max_pages):
    offset = 0
    pg = 1

    while pg <= max_pages:
        url = zone_url(zone_slug, offset)
        if not await safe_goto(page, url):
            break  # Pagina no cargada: salimos

        html = await page.content()
        listings, stats = parse_search_page(html)

        # Filtramos duplicados (puede haber solapamiento entre zonas)
        new = [l for l in listings if l["url"] not in seen_urls]
        for l in new:
            seen_urls.add(l["url"])

        # Verificamos si hay pagina siguiente
        soup = BeautifulSoup(html, "lxml")
        next_btn = soup.find(class_=re.compile(r"andes-pagination__button--next"))
        if not next_btn or next_btn.get("disabled"):
            break  # Ultima pagina

        offset = pg * 48 + 1  # ML usa 48 items por pagina
        pg += 1
        await asyncio.sleep(random.uniform(1.5, 3))  # Pausa entre paginas

Parseando HTML con BeautifulSoup

Una vez que Playwright nos da el HTML renderizado, necesitamos extraer la informacion. BeautifulSoup convierte el HTML en un arbol de objetos Python que podemos navegar facilmente.

Inspeccionando el sitio: el paso mas importante

Antes de escribir codigo, abre el sitio en Chrome y usa las DevTools (F12). El panel Elements te permite inspeccionar el HTML y encontrar los selectores CSS que identifican los datos que necesitas.

🔍
Flujo de inspeccion Click derecho en el elemento que te interesa → "Inspeccionar" → busca la clase CSS unica → usa esa clase en BeautifulSoup.

Parsear la pagina de listados

scraper/scraper.py 
from bs4 import BeautifulSoup
import re

def parse_search_page(html: str) -> tuple[list[dict], dict]:
    soup = BeautifulSoup(html, "lxml")  # "lxml" es el parser mas rapido

    # Encontrar todos los items de la lista (excluir banners/anuncios)
    items = [
        li for li in soup.find_all("li", class_="ui-search-layout__item")
        if "intervention" not in " ".join(li.get("class", []))
    ]

    results = []
    for item in items:
        # Extraer el link del anuncio
        link = item.find("a", href=re.compile(r"mercadolibre\.com\.uy"))
        if not link:
            continue  # Item sin link: skip

        url = link["href"].split("?")[0]  # Limpiamos query params de tracking

        # Extraer el precio usando el aria-label accesible
        price_span = item.find(attrs={"aria-label": re.compile(r"\d+\s+dolares", re.IGNORECASE)})
        if not price_span:
            continue

        price_match = re.search(r"(\d[\d.]*)\s+dolares", price_span["aria-label"], re.IGNORECASE)
        price = float(price_match.group(1).replace(".", ""))

        # Extraer la zona de la ubicacion
        loc_el = item.find(class_="poly-component__location")
        location = loc_el.get_text(strip=True) if loc_el else ""

        results.append({
            "url": url,
            "price_usd": price,
            "location": location,
            # ... mas campos
        })

    return results, {}

Usar regex para extraer numeros del texto

El HTML guarda numeros en texto plano como "42 m²" o "85.000 dolares". Usamos regex para extraerlos:

Parsear la pagina de detalle

Cada anuncio tiene una pagina propia con mas informacion. La tabla de especificaciones es clave:

scraper/scraper.py — scrape_detail() 
async def scrape_detail(browser, url, sem):
    result = {
        "gastos_comunes_uyu": None,
        "days_on_market": None,
        "m2_detail": None,
    }

    async with sem:  # Semaforo de concurrencia (ver capitulo 5)
        ctx = await new_context(browser)
        page = await ctx.new_page()
        try:
            if not await safe_goto(page, url):
                return result

            html = await page.content()
            soup = BeautifulSoup(html, "lxml")

            # La tabla "Caracteristicas del inmueble"
            table = soup.find(class_=re.compile(r"andes-table"))
            if table:
                for row in table.find_all("tr"):
                    cells = row.find_all(["th", "td"])
                    if len(cells) < 2:
                        continue
                    key = cells[0].get_text(strip=True).lower()
                    val = cells[1].get_text(strip=True)

                    if "gastos comunes" in key:
                        m = re.search(r"(\d[\d.,]*)", val)
                        if m:
                            result["gastos_comunes_uyu"] = float(
                                m.group(1).replace(".", "").replace(",", ".")
                            )

        finally:
            await ctx.close()  # Siempre cerramos el contexto

    return result

Paralelizacion con asyncio

Scrapeamos los detalles de 60 apartamentos. Si los procesamos uno por uno, con un promedio de 8 segundos por pagina, tardariamos 8 minutos. Con paralelizacion, podemos hacer 6 a la vez y terminar en 80 segundos.

Sincronico vs Asincrono

Async/await en Python: concepto

El modelo async de Python es cooperativo: una sola tarea se ejecuta a la vez en el thread principal, pero cuando una tarea necesita esperar (leer red, leer disco), le cede el control a otra.

concepto basico de async/await 
import asyncio

async def fetch_page(url: str) -> str:
    # "await" cede el control mientras espera la respuesta de red
    # Otro fetch_page puede ejecutarse mientras tanto
    response = await page.goto(url)
    return await page.content()

async def main():
    # asyncio.gather() ejecuta las coroutines en paralelo
    # Espera a que TODAS terminen y retorna una lista de resultados
    resultados = await asyncio.gather(
        fetch_page("https://ejemplo.com/1"),
        fetch_page("https://ejemplo.com/2"),
        fetch_page("https://ejemplo.com/3"),
    )
    # resultados = [html1, html2, html3]

asyncio.run(main())

Semaphore: controlando la concurrencia

Si abrimos 60 tabs al mismo tiempo podriamos: (a) agotar la RAM, (b) ser detectados como bot. Un asyncio.Semaphore limita cuantas tareas pueden ejecutarse simultaneamente:

scraper/scraper.py 
CONCURRENCY = 6  # Maximo 6 paginas abiertas al mismo tiempo

async def run():
    # El Semaphore permite que hasta CONCURRENCY tareas esten
    # dentro del bloque "async with sem" al mismo tiempo.
    # Las demas esperan su turno.
    sem = asyncio.Semaphore(CONCURRENCY)

    async def fetch_one(basic: dict) -> dict:
        # Esta funcion adquiere el semaforo antes de abrir el browser
        detail = await scrape_detail(browser, basic["url"], sem)
        return detail

    # Creamos una tarea por cada apartamento
    # gather() las ejecuta todas "a la vez" (respetando el semaforo)
    details = await asyncio.gather(*[fetch_one(b) for b in listings])
    # *[f(b) for b in listings] desempaqueta la lista como argumentos separados
como funciona el Semaphore internamente 
async def scrape_detail(browser, url, sem: asyncio.Semaphore):
    # "async with sem": intenta decrementar el contador interno del semaforo
    # Si ya hay CONCURRENCY tareas activas, ESPERA hasta que una termine
    async with sem:
        # --- Solo CONCURRENCY tareas pueden estar aqui al mismo tiempo ---
        ctx = await new_context(browser)
        page = await ctx.new_page()
        try:
            await page.goto(url)
            html = await page.content()
            return parse_detail(html)
        finally:
            # El "finally" garantiza que ctx.close() se llame aunque haya error
            # Al salir del "async with sem", el contador sube y otra tarea puede entrar
            await ctx.close()

Resumen del patron de paralelizacion

  1. Definir funcion async def fetch_one(item) que procesa un elemento
  2. Usar asyncio.Semaphore(N) como guard dentro de la funcion
  3. Lanzar con asyncio.gather(*[fetch_one(x) for x in items])
  4. El resultado es una lista en el mismo orden que items

Por que no usar threads?

Python tiene el GIL (Global Interpreter Lock): solo un thread Python ejecuta codigo a la vez. Para I/O-bound (esperar red), asyncio es mas eficiente que threads: no hay overhead de cambio de contexto y no necesitas manejar locks manuales. Los threads son mejores para CPU-bound (calculos intensivos).

Scoring y rentabilidad

El valor real del proyecto no esta en el scraping sino en la capa de analisis: convertir datos crudos en metricas que ayuden a tomar decisiones de inversion.

El tipo de cambio en tiempo real

Las propiedades se venden en USD pero los alquileres son en UYU. Para calcular la rentabilidad, necesitamos el tipo de cambio del dia:

scraper/scraper.py 
import requests

TC_FALLBACK = 43.5  # Valor por defecto si la API falla

def fetch_tc_usd_uyu() -> float:
    """Obtiene tipo de cambio USD/UYU de open.er-api.com (gratuito, sin clave)."""
    try:
        resp = requests.get(
            "https://open.er-api.com/v6/latest/USD",
            timeout=10,
            headers={"User-Agent": "Mozilla/5.0"},
        )
        resp.raise_for_status()  # Lanza exception si status != 2xx
        data = resp.json()
        tc = data["rates"]["UYU"]
        log.info(f"TC USD/UYU: 1 USD = {tc:.2f} UYU")
        return tc
    except Exception as e:
        log.warning(f"No se pudo obtener TC: {e}. Usando fallback {TC_FALLBACK}")
    return TC_FALLBACK  # Siempre devuelve algo valido

Formula de rentabilidad bruta

Formula

rentabilidad = (alquiler_mensual_uyu × 12) / (precio_usd × TC) × 100

Una rentabilidad bruta del 7% significa que en 14 años recuperas la inversion (sin contar gastos, impuestos, etc.).

scraper/scraper.py — build_apartment() 
def build_apartment(basic: dict, detail: dict, tc: float) -> dict:
    price = basic["price_usd"]
    zone = basic["zone"]

    # Estimacion del alquiler segun la zona (mercado local)
    estimated_rent_uyu = estimate_rent_uyu(zone)
    estimated_rent_usd = round(estimated_rent_uyu / tc)

    # Rentabilidad bruta anual
    rentability_pct = round(
        (estimated_rent_uyu * 12) / (price * tc) * 100,
        2
    ) if (price and tc) else 0

    # Precio por m2
    m2 = detail.get("m2_detail") or basic.get("m2")
    price_per_m2 = round(price / m2) if (m2 and m2 > 0) else None

    # Score: 10% de rentabilidad = 100 puntos
    score = calculate_score(rentability_pct)

    return {
        "price_usd": price,
        "rentability_pct": rentability_pct,
        "estimated_rent_uyu": estimated_rent_uyu,
        "estimated_rent_usd": estimated_rent_usd,
        "price_per_m2": price_per_m2,
        "score": score,
        # ... otros campos
    }


def calculate_score(rentability_pct: float) -> int:
    """Score 0-100. 10% anual = 100 puntos."""
    return min(100, max(0, round(rentability_pct * 10)))

Estimacion de alquiler por zona

Como los datos de alquiler no estan en ML (son ventas), los mantenemos manualmente en el codigo:

scraper/scraper.py 
# Diccionario de alquileres estimados por zona (UYU/mes, 1 dorm / monoambiente)
# Actualizar periodicamente con datos reales del mercado
ZONE_RENT_UYU = {
    "buceo":  27_000,
    "malvin": 25_000,
    "malvín": 25_000,  # con tilde
}
DEFAULT_RENT_UYU = 25_000  # Si la zona no esta en el diccionario

def estimate_rent_uyu(zone_str: str) -> int:
    norm = normalize(zone_str)  # Normaliza acentos y mayusculas
    for zone, rent in ZONE_RENT_UYU.items():
        if normalize(zone) in norm or norm in normalize(zone):
            return rent
    return DEFAULT_RENT_UYU


def normalize(text: str) -> str:
    """Elimina acentos y convierte a minusculas para comparacion robusta."""
    return (
        str(text).lower().strip()
        .replace("á", "a").replace("é", "e").replace("í", "i")
        .replace("ó", "o").replace("ú", "u").replace("ñ", "n")
    )

Cargar datos con Fetch API

El navegador carga el JSON directamente del mismo repositorio:

app.js 
async function loadData() {
  try {
    // Cache-busting: el ?_=timestamp evita que el navegador use
    // una version cacheada del JSON cuando los datos se actualizan
    const res = await fetch(`./data/apartments.json?_=${Date.now()}`);

    if (!res.ok) throw new Error(`HTTP ${res.status}`);

    const data = await res.json();
    allApartments = data.apartments || [];
    updateLastUpdated(data.last_updated);
    applyFilters();  // Aplica filtros y renderiza las cards

  } catch (err) {
    showError(err.message);
  }
}

// Llamado al cargar la pagina
document.addEventListener("DOMContentLoaded", () => {
  loadData();
});

Error CORS al abrir index.html directamente Los navegadores bloquean fetch() cuando abres un archivo HTML directamente (protocolo file://). Siempre usa un servidor local: python -m http.server 8080 y accede a http://localhost:8080.

Generacion de tarjetas (template literal)

En vez de un framework, generamos HTML con template literals de ES6:

app.js — cardHTML() 
function cardHTML(apt) {
  // Clase CSS segun el score para el color del badge
  const scoreClass = apt.score >= 65 ? "score-high"
                   : apt.score >= 40 ? "score-mid"
                   : "score-low";

  // Template literal: genera HTML como string
  return `
    <div class="apt-card">
      <img src="${esc(apt.thumbnail)}" alt="${esc(apt.title)}" loading="lazy">
      <div class="card-body">
        <div class="card-price">U$S ${Math.round(apt.price_usd).toLocaleString()}</div>
        <div class="score-badge ${scoreClass}">${apt.score}</div>
        <div class="stat-value">${apt.rentability_pct}% anual</div>
      </div>
      <a href="${esc(apt.url)}" target="_blank">Ver anuncio</a>
    </div>
  `;
}

// SIEMPRE escapar strings del usuario o externos para prevenir XSS
function esc(str) {
  return String(str || "")
    .replace(/&/g, "&").replace(//g, ">").replace(/"/g, """);
}
🛡
Seguridad: XSS (Cross-Site Scripting) Cuando insertas datos externos (titulos, URLs) en HTML, SIEMPRE usa la funcion esc(). Sin esto, un anuncio con titulo <script>malo()</script> ejecutaria codigo malicioso en el navegador del usuario.

function initDualRange(minId, maxId, fillId, fmt, minValId, maxValId) {
  const minEl = document.getElementById(minId);
  const maxEl = document.getElementById(maxId);
  const fill  = document.getElementById(fillId);

  function update() {
    const lo = +minEl.value, hi = +maxEl.value;
    const min = +minEl.min, max = +minEl.max;
    // Calculamos el % de posicion para el fill visual
    const pLo = (lo - min) / (max - min) * 100;
    const pHi = (hi - min) / (max - min) * 100;
    fill.style.left  = pLo + "%";
    fill.style.width = (pHi - pLo) + "%";
    // Actualizar labels
    document.getElementById(minValId).textContent = fmt(lo);
    document.getElementById(maxValId).textContent = fmt(hi);
  }

  minEl.addEventListener("input", () => {
    // Evitar que el minimo supere al maximo
    if (+minEl.value > +maxEl.value) minEl.value = maxEl.value;
    update();
    applyFilters();
  });
  maxEl.addEventListener("input", () => {
    if (+maxEl.value < +minEl.value) maxEl.value = minEl.value;
    update();
    applyFilters();
  });

  update();  // Render inicial
}

GitHub Actions: automatizacion

GitHub Actions es una plataforma de CI/CD integrada en GitHub. Nos permite ejecutar el scraper automaticamente todos los dias sin necesidad de mantener un servidor propio.

Conceptos de GitHub Actions

ConceptoQue es
WorkflowUn archivo YAML que define la automatizacion completa
TriggerEl evento que dispara el workflow (schedule, push, workflow_dispatch)
JobUn conjunto de pasos que se ejecutan en la misma maquina virtual
StepUn comando o accion individual dentro de un job
RunnerLa maquina virtual donde se ejecuta el job (ubuntu-latest)
ActionUn step reutilizable publicado en el marketplace (ej: actions/checkout@v4)

El workflow completo

.github/workflows/scrape.yml 
name: Actualizar apartamentos

on:
  # Cron: 8am hora Uruguay (UTC-3) = 11:00 UTC
  # Formato: minuto hora dia-del-mes mes dia-de-la-semana
  schedule:
    - cron: "0 11 * * *"
  # Permite disparar manualmente desde la UI de GitHub
  workflow_dispatch:

# El workflow necesita permiso para hacer commit y push
permissions:
  contents: write

jobs:
  scrape:
    runs-on: ubuntu-latest  # Maquina virtual gratuita de GitHub
    timeout-minutes: 30     # Matar el job si tarda mas de 30 min

    steps:
      # 1. Descargar el codigo del repo
      - name: Checkout repo
        uses: actions/checkout@v4

      # 2. Instalar Python 3.11
      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"
          cache: "pip"  # Cachea dependencias entre ejecuciones (mas rapido)
          cache-dependency-path: scraper/requirements.txt

      # 3. Instalar librerias Python
      - name: Instalar dependencias
        run: pip install -r scraper/requirements.txt

      # 4. Instalar el navegador Chromium para Playwright
      - name: Instalar Chromium
        run: playwright install chromium --with-deps

      # 5. Ejecutar el scraper
      - name: Ejecutar scraper
        run: python scraper/scraper.py

      # 6. Hacer commit de los datos actualizados y push
      - name: Commit y push
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add data/apartments.json
          # Solo commitea si hay cambios reales en el JSON
          if git diff --staged --quiet; then
            echo "Sin cambios. Nada que commitear."
          else
            git commit -m "chore: actualizar apartamentos $(date -u '+%Y-%m-%d %H:%M UTC')"
            git push
          fi

Cron syntax

cron format 
   ┌─ minuto (0-59)
   │  ┌─ hora (0-23, UTC)
   │  │  ┌─ dia del mes (1-31)
   │  │  │  ┌─ mes (1-12)
   │  │  │  │  ┌─ dia de la semana (0=Dom, 6=Sab)
   │  │  │  │  │
"  0  11  *  *  *  "   →  Todos los dias a las 11:00 UTC (8:00 Uruguay)

El archivo .nojekyll

Por defecto, GitHub Pages procesa el repo con Jekyll (un generador de sitios estaticos). Esto puede causar problemas con carpetas que empiezan con _ (como _next). Para deshabilitarlo, creamos un archivo vacio:

terminal 
touch .nojekyll
git add .nojekyll
git commit -m "chore: disable jekyll"
git push
